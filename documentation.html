Fensterchef

Layer

Fensterchef has three layers: The desktop layer at the bottom, the tiling layer and the floating layer at the top.

Let us break up all layers from bottom to top.

Desktop layer

The desktop layer is at the very bottom and meant to hold onto managed desktop windows. This can be windows holding icons and accepting input.

Tiling layer

The tiling layer is the most complex of all. It holds onto frames which cut the screen into pieces. Frames may be empty or contain a window.

Floating layer

The floating layer contains dock windows, fullscreen windows, dialog windows and regular floating windows. You can decide the size of regular floating windows and position them freely.

Window

A window is what you would expect! When starting a client, it may spawn as many windows as it likes. These can be windows our window manager should manage or unmanaged windows like popup menus of toolbars. Windows can be on any layer. You can toggle between the tiling and floating layer. There are 5 modes a window can be in.

Desktop window

Desktop windows are always on the desktop layer and you can only interact with their native interface. Our window manager does not mess with them too much.

Dock window

Dock windows are attached to the side of a screen. They may be bars or other attached windows.

Tiling window

Tiling windows are on the tiling layer and can be toggled to the floating layer. They are contained within frames and rigidly sized to fit within their frame.

Floating window

Floating windows are at the floating layer. They can be freely positioned and sized (depending on the size hints the underlying client configured for its window) and toggled to the tiling layer.

Frame

Frames are always on the tiling layer. The may be empty (ready to be filled with a window) or contain a window. Root frames are placed on each monitor. They can then be freely be split in two child frames without limit. When a frame is split, it becomes a parent.

Monitor

A monitor is directly correlated to a configured output device. xrandr is used to query the configured outputs. See the documentation for it.

Binding

Bindings are button or key bindings that can be associated to commands the window manager should run. These commands can be external shell commands or internal commands.

Default bindings

Use the the modifier key (default is Mod4 which is usually Super_L) together with below mentioned bindings.

LeftButton

Start moving/resizing the pressed window, this figures out dynamically which edge to move.

MiddleButton

Minimize (hide) the pressed window.

RightButton

Start moving the pressed window.

Shift + R

Reload the configuration file and parts of fensterchef.

A

Focus the parent frame.

B

Focus the child frame.

Shift + A

Focus the root frame.

Q

Close the current window.

- (minus)

Minimize (hide) the current window.

N

Put the next window into the current frame.

P

Put the previous window into the current frame.

R

Remove the current frame.

O

Replace the current frame with the last stashed frame.

Space

Move the focus from tiling to floating and vise versa.

Shift + Space

Make a tiling window floating and vise versa.

F

Put the current window into fullscreen or put it out of fullscreen.

V

Split the current frame horizontally.

S

Split the current frame vertically.

K/Up

Move the focus to the frame above the current frame.

H/Left

Move the focus to the frame on the left of the current frame.

L/Right

Move the focus to the frame on the right of the current frame.

J/Down

Move the focus to the frame below the current frame.

Shift + K/Up

Swap the current frame with the frame above it.

Shift + H/Left

Swap the current frame with the frame on the left of it.

Shift + L/Right

Swap the current frame with the frame on the right of it.

Shift + J/Down

Swap the current frame with the frame below it.

W

Show the interactive window list.

Return

Open a terminal window.

Control + Shift + E

Quit fensterchef.

Relation

A relation can be thought of as a connection between a window and commands. A relation is described using the window resource name and resource class (also known as window instance and class). To find out the instance and class of specific a window, inspect it using xprop. Look for the WM_CLASS property it. The first value is the instance and the second one is the class. By default there are no relations. You can freely configure your own.

Command line

Commands can be run externally in fensterchef using this command: 

fensterchef -e ...
A few examples: 
fensterchef -e 'focus right'
fensterchef -e 'run xterm'
fensterchef -e 'Mod4+Shift+a split vertically'
fensterchef -e 'unbind Mod4+Shift+a'
